#CS-002
The Fortress Gate 🏰
ArrayTwo Pointers Greedy20 ptsC (C11)

In the ancient kingdom of Aerondale, engineers are tasked with building a massive fortress gate between two defensive watch-towers.

You are given an array towers of length n, where each element represents the height of a tower built along a straight fortress wall. Each tower stands at position 0, 1, 2, …, n−1.

Gate Rules
  • You must choose exactly two different towers
  • The gate strength strength depends on both height and width
  • Find the maximum possible gate strength
Function Signature
int maxGateStrength( int* towers, int n);
Example
towers:[1, 8, 6, 2, 5, 4, 8, 3, 7]
Output:49
Best pair: i=1 (height 8), j=8 (height 7)
Strength = min(8, 7) × (8−1) = 7 × 7 = 49
Constraints
  • 2 ≤ n ≤ 10⁵
  • 0 ≤ towers[i] ≤ 10⁴
  • Return the maximum possible gate strength as an int

💡 HINT 1 — The O(n²) Trap
The most obvious approach is to check every possible pair of towers. For an array of n towers, there are n × (n−1) / 2 unique pairs. That gives you O(n²) time — and it works, but is there a smarter way?

Think about whether you really need to check all pairs, or if some pairs can be safely skipped without missing the maximum.
💡 HINT 2 — Think From Both Ends
Start by considering the widest possible gate — towers at positions 0 and n−1. This gives you the maximum width. Now you need to decide: can a narrower gate ever be stronger?
width decreases as pointers move inward
for strength to increase, height must compensate
→ use two pointers: one at each end, move inward
Two pointers, starting at both ends, moving toward each other in O(n).
💡 HINT 3 — Which Pointer Moves?
You have a left pointer and a right pointer. Each step you move one inward (sacrificing width). The gate height is min(towers[left], towers[right]).

Ask yourself: if you move the taller tower's pointer inward, the height is still capped by the shorter one — so width decreases but height cannot increase. What if you move the shorter one instead?
→ always move the pointer pointing to the shorter tower
→ moving the taller one can never improve the result
